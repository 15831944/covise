ANTIALIAS <flag>

Enables or disables antialiasing. If ANTIALIAS is ON but the graphics board doesn't support multisampling antialiasing is not enabled. If the line is ommited but the graphics board supports multisampling antialiasing is enabled. If ANTIALIAS is OFF the frame buffer is not configured for multisampling. 
    ANTIALIAS OFF

ARENA_SIZE <size>

 sets the size of the shared arena Performer uses. The default size is 256 MB. The size is defined in byte. If you want to increase the shared arena size please read the man pages of pfSharedArenaSize before. Example:

increase the shared arena size to 1000 MB: 

    ARENA_SIZE 1000000000

AUTO_WIREFRAME <flag>

 enables or disables auto wireframe mode. auto wireframe means that all scene objects are drawn in wireframe mode as soon as the user selects the menu. Possible values are ON and OFF. If you omit the line it is OFF.

BUTTON_SYSTEM <system>

 specifies the use of an extra button devices which doesn't come with the tracking system. For example the Motionstar system from Ascension comes with a 3D mouse but DVISION offers an own 3D device where the scension sensor is integrated. For using the ascension mouse you do not have to specify a button system but for using the DVISION mouse you have to specify it in CoverConfig. The device is further configured in the section ButtonConfig.

Supported systems are

  • IAO
  • CAVELIB
  • DVISION ???
    • Please contact us before you want to use one of the above 3D devices.

    COORD_AXIS <flag>

     ON/OFF enables or disables drawing of an axis system in the renderer window at starting time. If the line is omitted the coordinate axis is displayed at starting time.

    FPS <flag>

     ON/OFF enables or disables printing the frame rate (FPS = frames per second) into the terminal window. If the line is omited printing the frame rate is disabled.

    Example: 

        FPS OFF

    WELLCOME_MESSAGE <message>
    FOOTER_MESSAGE <message>

    If a wellcome message is specified this message is displayed for some seconds when COVER is started. If a footer message is specified, this message is always displayed at the bottom of the renderer window. Please note that the text is drawn only in the stereo channel of the right eye.

    Example: 

        WELLCOME_MESSAGE Wellcome to COVISE-VR
    FOOTER_MESSAGE Data courtesy of ...

    FREEZE <flag>

     flag=ON/OFF enables/disables headtracking at starting time. If the line is omitted headtracking is ON at starting time.

    Example: 

        FREEZE OFF

    LIGHT1 <xpos> <ypos> <zpos> <w>
    LIGHT2 <xpos> <ypos> <zpos> <w>

     You can define 2 additional lights. xpos, ypos and zpos define the position and w specifies, if the light is local or infinite

    Example: 

        LIGHT1 0 10000 10000 1.0

    MENU_POSITION <x> <y> <z>

     specifies the position of the menu in the world coordinate system. The default position (if the line is omitted) is x=0.6*sceneSize/2.0, y=0.0, z=0.6*sceneSize/2.0.

    Example: 

        MENU_POSITION 900 0 800

    MENU_ROTATION <angle> <x> <y> <z>

     specifies the orientation of the menu in the world coordinate system. [x y z] define an axis and angle is the rotation in degree around this axis. The default orientation (if the line is omitted) is angle=-30 axis=[0 0 1]. This is a rotation around the z axis.

    Example: 

        MENU_ORIENTATION -30 0 0 1

    MODELPATH <path>
    MODELFILE <file>

     COVER is based on the graphics library IRIS Performer. Performer is able to load many 3D file formats like obj, dxf, iv, etc.. The MODELPATH and MODELFILE entries are a way to load models into the renderer without using COVISE. It works if COVER was started inside COVISE or in standalone mode. MODELPATH defines the absolute path to a directory with model files, MODELFILE specifies the file.

    Example: 

        MODELPATH /usr/share/data/models
    MODELFILE banana.iv

    MONO_COMMAND <command>
    STEREO_COMMAND <command>

     For stereo projection the application has to draw two images and the display system has to be stereo capable. There are two ways to display the stereo images: 1. display the left and right eye view one after another (shutter glasses mode) or display them at the same time with two polarixed projectors (passive stereo mode with polarized glasses). COVER supports both modes. This sections is about the shutter glasses stereo mode.

    COVER always starts in stereo mode this means it draws two images. If the video format is a stereo format (for example 1024x768_96s) then you will automatically see two images without setting STEREO_COMAND.

    In the case that the system has a mono video format at starting time you can provide a command to switch to stereo for example /usr/gfx/setmon -n 1024x768_96s. If you want to go back to mono after COVER is finished you can provide a mono command. Please note that not all video formats can be downloaded with setmon. Read the man pages about setmon and ircombine.

    MULTIPROCESS <flag>

    IRIS Performer can fork separate processes for application, culling and drawing. With MULTIPROCESS ON you enable the multiprocessing modell APP_CULL_DRAW (see man pfMultiProcess). On multiprocessor systems multiprocessing should be enables. On singleprocessor systems and for debugging purposes multiprocessing can be switched off.

    Example: 

        MULTIPROCESS ON

    NOSEPARATION <flag>

     For some purposes like making photos or images you don't want to see the two images. If your projection system is configured for a mono mode you can simply switch to mono through the 3D menu in COVER. In the case your projections system is configured only for stereo video formats you have to can't switch to mono mode. The solution is to set the stereo separation to 0 ot to enable NOSEPARATION

    Example: 

        NOSEPARATION ON

    NOTIFY <mode>

     sets different levels of Performer debugging messages. Valid values are
  • OFF or line omiteed- sets the notify level to PFNFY_FATAL
  • ON - sets the notify level to PFNFY_DEBUG
  • NOTICE - sets the notify level to PFNFY_NOTICE

    • Example: 

        NOTIFY OFF

    SPOTLIGHT <flag>

     enables/disables the spotlight at starting time. The spotlight can be switched on/off also through the 3D menu in COVER.

    Example: 

        SPOTLIGHT OFF

    TWOSIDE <flag>

     enables/disables twosided lighting. Twosided lighting is a performance penalty but necessary if some faces of a model have a wrong orientation or if inner sides should be lighted. For COVISE geometry twosided lighting is recommended. If this line is ommited twosided lighting is ON

    Example: 

        TWOSIDE ON

    TRACKING_SYSTEM <system>

    This parameter describes the tracking system type. Supported systems are:
    • POLHEMUS (FASTRAK with Stylus from Polhemus)
    • FLOCK (Ethernet Motionstar with 6DOF mouse from Ascension)
    • LOGITECH (not supported any more)
    • SPACEPOINTER (2D mouse steering)
    • SPACEBALL (SPACEMOUSE from DLR)
    • BEEBOX (undocumented)
    • FLYBOX (undocumented)
    • PHANTOM (undocumented)
    Example: 
        TRACKING_SYSTEM POLHEMUS
    If the keyword TRACKING_SYSTEM is missing a default tracking system is used. The default tracking system is SPACEPOINTER.

    The systems POLHEMUS and FLOCK rquire further configuration in the section PolhemusConfig and FlockConfig. Please note that in COVISE 4.5.2 most of the configuration parameters of FLOCK have to be in the section PolhemusConfig. This will change in the next release.

    SCENE_SIZE <size>

    The scene size is used for scaling objects into the visible area. Therefore the scene size should be in the same range as the screen size is. It is measured in mm.

    Example:
    Screen size of the wall of the RUS CUBE is 2800 x 2500 mm 

        SCENE_SIZE 2500
    Screen size of a 19" monitor is 340 x 270 mm 
        SCENE_SIZE 270

    NUM_PIPES <num>

     defines the number of grafics pipes you want to use for your application. It doesn't have to be equal to the number of pipes in your machine.

    Example:
    The Onyx which drives the RUS CUBE has 3 graphics pipes but only 2 are used for the CAVE: 

        NUM_PIPES 2

    NUM_WINDOWS <num>

     defines the number of windows. In a 4 wall cave system you have 4 windows, on the Holobench you have 2 windows and on an Immersadesk or Workbench you have 1 window.

    Example:
    The RUS CUBE has 4 walls: 

        NUM_WINDOWS 4

    STEREO_SEPARATION <separation>

    sets the separation between the eyes and is measured in mm. Here a good rule to compute a reasonable separation: 
        sep = 2 * opt_dist * tan( betamax / 2 ) * ((i + 1) / i)
    opt_dist stands for the optimal viewer distance when the field of view is 45 degrees: 
        opt_dist = scene_size / (2 * tan (22.5)) 
    betamax = 1.73 degree
    tan(betamax/2)=0.015098245
    i=yfar/opt_dist
    Example: 
        scene_size = 2500
    opt_dist = 3017.767
    yfar = 2 * scene_size
    i = 5000/3017.767=1.656
    sep = 2*3017.767 * 0.015098245 * 1.6= 145.8

    RELATIVISTIC_PARAMETER_FILE <file>

     is the location of an additional parameter file with relativistic options. The syntax of this file is: 
      dummy_value 
  start_x start_y start_z 
  direction_x direction_y direction_z
  begin_time delta_time end_time
  dummy dummy dummy
  num_lights
  { light_x light_y light_z }
  overall_scale
  acceleration_scale_x 
  acceleration_scale_y 
  acceleration_scale_z
    start_? is the initial position of the observer with respect to the frame of reference of the objects.

    direction_? is the initial direction of the motion of the observer with respect to the frame of reference of the objects.

    begin_time, delta_time, and end_time describe the propagation of the proper time of the observer.

    num_lights is the number of lights. The direction of the lights {light_x light_y light_z} has to be specified num_lights times.

    overall_scale is the overall scaling of the three-dimensional model.

    acceleration_scale_? is the sensitivity with respect to accelerations.

    RELATIVISTIC_Mode <flag>

    is ON or OFF. Default is off.
