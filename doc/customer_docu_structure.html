Documentation Structure for Customer Modules

(Reference Guide and Online)
 

In order to provide (printable and online) documentation for user-specific modules (currently for sca) the following framework has been provided in ClearCase (to be merged):
 
 
 

For printable docu:


doc
    |
customerguide
    |    Makefile
    |    customerguide.tex
    |    etc.
modules
    |    modules.tex
Customer
    |
Test
        Test.tex
        Test.gif
        Test.eps
 
 
 
 

For online docu:


doc
   |
online
   |
modules
   |
Customer
   |
Test
        Test.html
        etc.
        (generated by converter!)
 
 
 
 
 
 
 

For accessing online docu via help:


doc
   |
help
        start_overview_cust.html replaces  start_overview.html
        helpmodules_gen_toc.html replaces helpmodules_toc.html
        customer.html
        customertoc.html
        start_customer.html
        helpmodules_cust_toc.html
 
 

In order to add a customer module to printable docu, please perform the following steps:


1. Add directory and.tex file (plus .gif and .eps) in doc/customerguide/modules/Customer (like 'Test')
2. Add module name in doc/customerguide/modules/modules.tex (and comment out 'Test')
3. Test your latex docu using:
latex customerguide.tex
xdvi customerguide.dvi
4. If you have documented all of your modules:
Generate the complete 'Customer Module Reference Guide' using 'make customerguide'.
(you can print the .pdf like the Module Reference Guide)
 
 

In order to generate online docu (after the printable docu has been tested):


'make customerhtml'
(and - for testing - view the result with
 netscape doc/online/modules/Customer/modulename/modulename.html)
 

In order to access the online docu via online help (will be probably my responsibility)


1. Insert the module in the table of contents for customer modules   (doc/help/customertoc.html) including link
2. Insert a short description of the module in customer.html
3. Make sure that start_overview.html has been replaced by start_overview_cust.html
(For testing: Select 'Help --> Modules' and replace in netscape manually!)
4. Select help 'Help --> Modules --> Customer --> module name
5. Insert link to module docu in short description (customer.html)
 

Note:


You (the module owner) will provide the directory plus .tex, .gif and .eps files for the module and modify modules.tex; I will modify customer.html and customertoc.html accordingly and test it.
The rest of the framework is already provided and has only to be activated replacing  start_overview.html by start_overview_cust.html and helpmodules_toc.html by helpmodules_gen_toc.html.
 
 
 
 
 

Open question:


If you select the module icon and click on 'help', you are told 'no documentation available' but  you get the Module Reference frame and you can select 'Customer --> module name' and actually get your module documentation. To get it directly, a change in the MapEditor may be necessary.
