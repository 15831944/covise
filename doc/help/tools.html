Module Reference Guide

Tools

Note:
If you are looking for a module you might have problems starting with the right category. Strictly speaking, 'Filter' (pass only selected information) and 'Mapper' (create an image out of data) are subsets of 'Tools'; so if you don't find a suitable module in 'Filter' or 'Mapper', look up in 'Tools', and vice versa! (use Overview - you will find there also an alphabetical module list) 

    AssembleUsg
    converts blocks of unstructured grids (per timestep) to one unstructured grid (per timestep) without any block information. This module has been developed to use the module TetraTrace also for blocks of grids.

    BlockCollect
    collects different objects and puts them into one set.
    You can choose among three different rules to build up this set:

      1. Set of Elements
      2. Cat Timesteps
      3. Merge Blocks


    BoundingBox
    writes a message with information about the smallest parallelepiped parallel to the globale coordinate system which contains an unstructured grid (or set of grids).

    Calc
    allows you to perform mathematical calculations with scalar and/or vector objects. The result may be a scalar or a vector. The module interprets a string according to a syntax described below.

    CellToVert
    interpolates per-cell data to per-vertex data
    (Simulation codes like Fluent or StarCD compute results per-cell. Most COVISE modules expect per-vertex data.)

    CheckObj
    tests the object received at the input port (unstructured grid, polygons, lines, triangle strips).

    Collect
    combines geometric primitive objects like points, lines, polygons, triangle strips with colors, normals and textures.

    CropUsg
    crops the parts of an unstructured grid which are outside of a bounding box.

    CutGeometry
    cuts geometry objects. The part which should be cut away is defined through a plane. This is usually used to make the inner parts of geometry objects visible.

    CuttingSurface3DTex
    combines a cuttingsurface and colors on a cuttingsurface with an uniform grid and color on a uniform grid into a set of two geometry objects. The uniform grid and the colors on the uniform grid represent the voxels of a 3D texture.

    DerivOperators
    works out gradients of scalar fields, or the divergence or curl of vector fields.

    DisplaceUsg
    displaces an unstructured grid by using displacement vectors for every vertex.

    DomainSurface
    computes the outer surface or edges of an unstructured grid.

    DomainSurfaceStr
    computes the outer surface or edges of a structured grid.

    FixPart
    fixes one part in a simulation over timesteps and moves all other parts relativ to the fixed one.

    FixUsg
    removes double and unused vertices.

    GenNormals
    generates normals for a set of polygons.

    GetSetelem
    performs the operation of selecting an element out of a set.

    GridSurface
    analyzes grids and generates surfaces as polygons. It shows all surfaces,  i.e. including the inner ones.

    Histogram
    generates a histogram from a data set, i.e. counts for every data value how often it appears in the data set.

    ImageToTexture
    generates a texture for a polygon input object given a TIFF file. The default mapping that relates the geometry point
    coordinates with the texture coordinates is a simple projection onto the XY plane. You may introduce your own mapping if the module
    gets a vector field as input object.

    Interpolate
    creates a set of unstructured vector or scalar data by interpolation of the incoming data. For each point of the data set the intervall between zero and the current value is decomposed into a given number of segments.

    MakeOctTree
    generates octtrees for unstructured grids and polygons.

    MakePlots
    generates a set of DO_Unstructured_2D_Data objects, which may be  plotted by Plot (see Renderer modules) out of some scalar fields.

    MakeTransient
    allows you to generate a specified number of timesteps (default 72) out of 'static' data. With a Timestep Sequencer that shows up in the Renderer you can go thru your timesteps, either step by step or continuously using 'play'.

    Material
    contains a list of different color values from which the user can select.

    MinMax
    calculates the minimum and maximum value of a scalar dataset. It outputs either a min-max pair that can be used for example as input for a colormap module, or a list of x-y pairs that shows the distribution of values over the range.

    PartSelect
    is able to manage sets to cull parts of the geometry or data based on the part attribute.

    PipelineCollect
    reduces the maximum usage of memory by executing a pipeline stepwise.

    Sample
    transforms an unstructured grid and data on the unstructured grid into a uniform grid and data on the uniform grid. As the uniform grid has the extension of the bounding box around the unstructured grid, data on parts of the uniform grid which are outside the unstructured grid are either set to a certain value or max_float. The module Colors computes a transparent color for MAX_FLT data so those parts appear transparent in the Renderer.
    A uniform grid and colors on the uniform grid can also be seen as a 3D texture. If you cut a plane out of the 3D you get a 2D texture (an image). Mapping this texture on a plane gives similar information as a cuttingsurface through the 3D grid with data mapped on the surface as colors.
    3D textures are supported by Octane and Onyx systems and are computed by the graphics hardware and are thus very fast.

    Scalar2Vector
    combines scalar data given at the input ports to vector data. If input data have attributes they are summarized where it makes sense.

    ShowFaceNormal
    computes the normal of a polygon or a triangle strip.

    ShowGrid
    for structured grids: computes the grid lines of the grid.
    There are different display methods:

    • show all grid lines
    • show the grid lines on the outer surface (hull) only
    • show three sides of the hull only (the user may choose among 8 options that determine which three sides are selected)
    • show only the bounding box only
    • show the edge lines only
    for unstructured grids: displays only one cell; the cell is selected through the parameter position.

    Solutions
    calculates Plot3D solution data (data read by ReadPlot3D).

    Sphere
    builds spheres (made up of triangle-strips) out of points (e. g. retrieved using a particle tracer).

    StoU
    transforms a rectiliniar, structured or an uniform grid received to the inout port into an unstructured grid which can be composed of tetrahedrons, hexahedrons, pyramids or prisms.

    TensorScal
    extracts invariants from tensor fields.

    Tetrahedronize
    converts unstructured mixed-cell-type grids into tetrahedra grids.

    Transform
    manipulates grids and geometry objects. Transform makes it possible to mirror, translate, scale, rotate, and tile objects and their data.

    Tube
    creates cylinders around lines. The shape of the tubes depends on the radius, number of sides (scalar parameters) and the type of the cylinder (open, closed or arrow).

    VectorScal
    extracts scalar values from vector fields.
