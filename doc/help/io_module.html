Module Reference Guide

IO_Module:
GenDat
creates a chunk of a regular grid. It generates meshes based on carthesian and cyclic coordinates. It is mainly used as a generator of test and reference data.

PerformerScene
performs fake reading for Performer Models.

ReadABAQUS
reads the results from ABAQUS computations as .odb files.

ReadANSYS
reads .rst, .rfl, and .rth files, which are the result files of mechanical, FLOTRAN, or thermal analysis. In all cases READANSYS finds dynamically the degrees of freedom active in the model, from which the user may choose any option for visualisation. As for derived data, the module reads stresses and several kinds of strains for mechanical files, and the heat flux for thermal analysis.

ReadCadmould
reads in Cadmould files.

It reads in the mesh and may get information from

    • the rheology file (<basename>E0),
    • thickness of the outer layer (<basename>S##),
    • magnitudes related to the filling process (<basename>a), and
    • data from the intermediate state files (<basename>E##).
More over the module reads information from files in the CAR format.

ReadCFX
reads CFX-5 data format.

ReadDx
reads Data Explorer Ascii files - experimental version, use at your own risk; docu not yet complete.

ReadDyna3D  (SGI only - not supported on Linux)
reads the binary plot file "LS930.PTF" written by LS-DYNA3D, a software package for Nonlinear Dynamic Analysis from Livermore Software Technology Corporation. Attention: Thick shell elements are not supported!

ReadEnsight
reads data files written by Ensight (versions 5.0 and gold).

Ensight 5.0 data consists of files:

    • description of the geometry (*.geo) - required
    • result file (*.res) - optional
    • variable files  - optional
Ensight gold data consists of files:
    • description of the geometry (*.geo) - required
    • case file (*.case) - required; in this file are pointers to all other necessary files.
Timesteps are supported.

ReadEnsightNT
reads data files written by Ensight. For supported versions see table in module documentation.
Note: ReadEnsightNT is planned to replace ReadEnsight as soon as it will accept all Ensight formats.

ReadFidap
reads data from Ihs FENFLOSS.

ReadFloWorks
reads FloWorks files - experimental version, use at your own risk; docu not yet complete.

ReadFluent
reads formatted Fluent(R) files V 0.99b.

ReadIv
reads Inventor files. You have to use the Open Inventor Renderer to display the file.

ReadKiva  (SGI only - not supported on Linux)
reads data sets written by  KIVA.

ReadLat
reads the ASCII latice file written by IRIS-Explorer.

ReadMovieBYU
reads MovieBYU data in ascii format. It supports scala, vector and timestep data additional to the grid data. Optional is a color file in ascii format with SGI colors.

ReadNastran
reads data written by MSC Nastran in binary output2 Version 70 format. It currently supports a subset of the NASTRAN data format.

ReadObj
reads Wavefront OBJ files.

ReadPAM  (SGI only, PAM license required - not supported on Linux)
reads data sets DSY and THP files. From the DSY file it may read the grid, nodal and cell data. Global data may be read from either the DSY or the THP file.

ReadPatran
reads data sets compatible to  PATRAN 2.5 file format. It processes information of the neutral file, displacement or force result files, nodal and element result files.

ReadPlot3D  (SGI only - not supported on Linux)
reads data sets written by Plot3D.

ReadStar  (SGI and Linux - since COVISE Rel. 5.2.3)
reads data sets written by Star-CD versions 2.21 thru 3.1b. It needs a ProSTAR input file (*16, *mdl) and a result file, either stationary (*09/*pst) or transient (*29/*pstt).

ReadStl
reads Sandia Stl data.

ReadTascflowTDI  (SGI and HP only - not supported on Linux)
reads data sets written by CFX-TASCflow versions 2.7-2.10. It needs the information contained in the grid (GRD), block-off and boundary condition (BCF) and results (RSO) files.

ReadTecplot
reads Tecplot V 1.0b files.

ReadVolume
reads volume data files, and provides the scalar volume data as a uniform grid.

ReadVTF
converts the representation of a surface in ViewTech File format (specified as file browser parameter) to a format acceptable for COVISE (on the first output port); in addition, the user can choose between scalar and vector results, which are provided by the chosen file. These results can be found at the last four output ports.

RW_ASCII
reads or writes COVISE ASCII files.

RWCovise
reads or writes all data objects that are available in  COVISE.

RWCoviseGroup
reads or writes a group of COVISE files.

WriteASC
generates an ASCII file containing the information of a Covise object (accepted types are specified in the list of types of the input port). You may use this file for reporting or in case you wish to analyse the data with yout own tools.

WritePatran
writes a geometry in Patran Format - transforms a surface in COVISE format to Patran format.

WriteVolume
writes volume data files from scalar volumetric data on uniform grids.
