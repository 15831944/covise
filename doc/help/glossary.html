Glossary

called by a term in italics, e. g. Antialiasing or by Glossary
(see  Module Reference Guide  for information about modules)

Alpha blending
Additional information for each pixel for creating transparent materials

Aliasing
- the familiar "staircase effect". Jagged transitions are often formed between adjacent pixels in the representation of diagonals or curves. These "jaggies" can be smoothed out by anti-aliasing.

Antialiasing
This feature eliminates or reduces jagged lines and makes objects drawn on the screen look smoother.
Warning: Enabling Antialiasing reduces drawing speed!

Ambient color
reflected color of an object in response to the ambient lighting in the scene

BASE COLOR lighting model
Each object has its own base color also called diffuse color; objects are rendered by taking in account their diffuse color only (no shading) - see also PHONG lighting model

Clipping Plane(s)
You cut off part of your geometry object(s) by one or two planes
COVISE provides three possibilities:
Clipping Plane (Renderer, menu item Viewing > Clipping Plane)  1 switchable plane with variable normal and a variable point on the plane
Clipping Planes ( PreferenceSheet) 2 switchable clipping planes with fixed normals, distance varied by a wheel
CutGeometry (Module) 1 permanent plane with variable normal and a variable point on the plane
(to be supplied as module parameters)

Control Panel
The Control Panel is created when COVISE starts and appears initially as an icon only. Its purpose is the collection of graphical interactors which typically represent often used and changed parameters, and it is opened when such an interactor is loaded.

COVER (Renderer)
COVER stands for "COVISE Virtual Environment Renderer" (also called "VR Renderer"), and is an integral part of the COVISE visualization and simulation environment. The design of COVER was developed to support technical and scientific applications. The rendering of COVER is based on IRIS Performer to make optimal use of the SGI rendering hardware.
(also called VR Renderer - see also OpenInventor Renderer, IV Renderer or Desktop Renderer)

COVISE
COllaborative VIsualization and Simulation Environment - extendable distributed software environment to integrate supercomputer based simulations, postprocessing , multimedia and visualization functionalities with cooperative working in a seamless manner

COVISE Request Broker (CRB)
COVISE data management process

Desktop Renderer
see Open Inventor Renderer

Diffuse color
object's base color (specified in the color array of the renderer geometry input data objects)

Directional light
This light illuminates uniformly along a particular direction.

Discrete mesh
draw style: dark texture on bright visible surfaces only

Double buffering
Two display buffers are used. This means that the next image can be drawn in the page of the display buffer, which is initially invisible. This image will be be displayed once it is ready and the next image will be prepared in the other page of the buffer. Animations and games can be made to look more realistic with this technique than with simple single buffer.

Emissive color
color of light caused by self illuminating object

Hidden line
draw style: bright texture, visible lines only

Inventor Renderer (IV Renderer)
see Open Inventor Renderer

Map
Dataflow network for postprocessing and visualization of simulation data; this network starts usually with one or more I/O (Read) modules that accept the simulation data in COVISE format (e. g. RWCovise), modifies the geometry by Filter modules (e. g. CutGeometry), generates surfaces with equal values by Mapper modules (e. g. Isosurface), maps data to color by Color modules (e. g. ColorEdit), generates particle traces or streamlines by  Tracer modules (e. g. TetraTrace),  and ends with one or more Renderer or other output modules to visualize the results

Map Editor
Motif based user interface to build a map i. e. a dataflow network for postprocessing and visualization of simulation data, and to execute this map. You start it by typing 'covise'. Messages from the MapEditor
begin with UIF (User Interface)

OpenGL
3D software interface (3D API). Based on Iris GL from Silicon Graphics.

OpenInventor Renderer (IV Renderer)
COVISE Renderer based on the OpenInventor graphics toolkit of SGI using OpenGL as its rendering interface - called 'IV Renderer', 'Desktop Renderer', or 'Renderer' for short (but see also COVER Renderer)

PHONG lighting model
This light model takes into account all light sources in the scene and the object's surface orientation with respect to the lights to generate a realistic smooth shaded 3D appearance of an object (see also BASE COLOR lighting model).

Point light
A point light, like a star, radiates light equally in all directions from a given location in 3D space

Renderer
Renderer is a category of output modules. In the context of Online Help, it is often used as synonym for the OpenInventor Renderer or IV Renderer (see above).

Shutter glasses
Goggles which use stereoscopic LCD projection of 3D scenery to give the observer a strong impression of space

Single buffer
By contrast with double buffering, where the image buffer is duplicated, the single buffering mode is not able to access the next image, which has already been calculated. This means that animations will run jerkily.

Smooth normals
You approximate a surface by a polygon; if you use one normal per vertex you get a smoother surface than if you use one normal per polygon face

Specular color
color on highly reflecting parts, i. e. color of the incident light

Spot light
A spot light illuminates from a point in space along a primary direction. Its illumination is a cone of light diverging from the light's position.

Textures
Wrapping a bitmap around an object, including perspective correction, for example wallpaper on a wall or a wood texture.

Transparency, blended T.
uses GL alpha blending
Transparency, delayed blended T.
uses GL alpha blending; opaque objects are rendered first, then transparent objects
Transparency, screen door T.
This is the default and the only supported mode on some machines. It uses GL patterns for achieving the transparency effect.
Transparency, sorted blended T.
uses GL alpha blending; opaque objects are rendered first, then transparent objects; additionally objects are sorted by their distance from the camera and are drawn from back to the front

Transparency level
If you have two objects, one covered by the other, the transparency level means
           0.0 no transparency you see the covering object only
0.0< transparency <1.0 limited transparency you see both objects
             1.0 full transparency you see the covered object only

VR Renderer
see COVER

Wireframe
draw style: complete grid (bright) including hidden lines
