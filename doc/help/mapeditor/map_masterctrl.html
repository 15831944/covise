Note 2:
You can use this (and no other) menu item as Slave only!

A master/slave relationship is applied among participating partners of a session.
This means, that at every point in time only one participant has control over the overall session, while the others can only watch ongoing actions performed by the master. All slave windows are insensitive for user input. But all important events are spread to all slaves. Therefore when a new map is generated or editied, every module icon and connection line between module ports also appear in the slave MapEditor working area. The modifications in the Module Setup and attached parameters in the Control Panel are also spread to the slave sides.

Changing the roles of Master and Slave works as follows:

  • As slave partner you can use only the menu item Master Control (appears in black =  selectable; all other menu items appear in grey = not selectable)

    •  
  • Selecting Master Control you send a Master Request to ask for master status

    •  
  • A question dialog (see figure below) pops up on the master side
  • If the master replies No you will be informed by
  • If the master replies Yes
    • all previously grey menu items on your side are now black (selectable), only  MasterControl becomes grey (not selectable)
    • all previously black menu items on the master side are now grey (not selectable), only MasterControl becomes black (selectable)
Note 3:

Please observe the following restrictions for scalar/slider parameters if their appearance type has been changed to player/sequencer:
    • a player/sequencer will be stopped if you transfer control to the slave
    • the "delta value" will not be transferred to the slave; so if you don't like the default value you have to explicitly set delta at the slave again
