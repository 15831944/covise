Working with Modules

Short Information about Modules:

You get
module information (function and parameters / input data / output data) in a 'book with 3 pages', the Module Information window by
clicking on the red book icon of the module with the left mouse button

Other Operations with Modules:

You can perform some
module operations, called Further Actions by
clicking on the module icon (anywhere) with the right mouse button to get a selection window
selecting the operation with the left mouse button

 

 These operations are

  • Execute

    • You execute the map starting with this module, not from the beginning. It is typically more efficient to execute only a part of the module network after having changed some parameters instead of executing the whole network. I/O modules often need a lot of time to read in large data files which is not necessary if you have just modified a module parameter further down in the processing chain.

  • Delete

    • You remove a module from the working area.
     
  • Rename

    • Rename a module (also used to generate a label for a module group)
     

    If you press 'OK', 'DomainSurface' will be changed to 'Oberfläche'.
     
  • Move

    • You move this module to another host for execution. This is only possible, if an additional host has been added to the session via 'Add Host/Partner' (see menu option CSCW).
     
  • Copy

    • You copy this module to another host for execution. This is only possible, if an additional host has been added to the session via 'Add Host/Partner' (see menu option CSCW). The Mapeditor also copies the current values of the parameter and connection lines to other modules.
     
  • Help

    • You get the complete documentation about this module from the Online Module Documentation (if not available, use the ModuleInformation window to get short information for this module or use  Modules to get information  for all modules)
