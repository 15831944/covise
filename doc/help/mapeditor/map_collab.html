Distributed and Collaborative Working

Here you get some background information on the COVISE architecture, and you will see the difference between distributed and collaborative working; for more information see Tutorial (Advanced Topics) and User's Guide, chapter COVISE CE.

Distributed Working
 

The figure above shows the elements of distributed working in COVISE. The application consists of three modules:

    • a module which reads in data (READ)
    • a module which extracts a special feature (FILTER) and
    • a module which displays the extracted data (Renderer, RENDER).
As the filter module consumes much CPU time and memory it will be started on a remote compute server.
The first process which is started when 'covise' is typed in is the Controller which in turn starts the interface process (MapEditor) and the data management process (CRB, COVISE REQUEST BROKER). As soon as another host is included in the session a CRB is started on that computer. THe read module is started on the local workstation, the filter module on the remote computer.
