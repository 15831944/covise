Execution

 

Execution provides the following options

  • Execute

    • executes the complete map in any case, independent of the previous state, from the beginning; stored parameter changes are applied.
    Notes:

    Same function (execute from the beginning) available via
      • Exec Icon (   ) from the Tool Bar (shortcut).
      • Execute all from the ControlPanel (index card)

        •  
    To execute the map starting with a specific module use
      • Execute in the menu Further Actions (Working with Modules)

        •  
      • Execute from the Module Information window 

        •  
      • Exec icon (  ) from a module name with a parameter interactor, if available, in the  ControlPanel (index card)
  • OnChange

    • executes all modules of the map where changes apply or which are influenced by changes in other modules. It initiates an execution after every single change. Player and Sequencer in play or reverse play mode (not in single step) will automatically set OnChange and start the execution of the map repeatedly until their ending condition is met; you can also set OnChange explicitly from the ControlPanel

    Note:
    You can follow up the flow of control thru the map: the frame of the module just executed turns red.
    (default - can be switched off via Covise config.file if necessary for performance reasons)

If the map contains a renderer and the pipeline is executed successfully, the resulting geometry appears in the in the viewer area of the Renderer. In addition
    • all geometry objects shown in the viewer area are listed in the GeometryObjects list box of the Renderer
    • all color maps used in the viewer area are listed in the ColorMaps list box of the Renderer
    • data objects used in the map are updated in the COVISE Objects (DataObjectList) index card of the MapEditor
