CSCW (Computer Supported Collaborative Work) - AddHost/Partner

Notes:
  • You can use this menu item as Master only!

    •  
  • It is recommended that all collaborating hosts have COVISE version 5.2 (or higher) installed! 

    • Otherwise not all options may be available and you may get compatibility problems.
     
  • For a more comfortable way to establish working groups of users ("rooms") see "New Collaborative COVISE (Daemon)"  in  User's Guide, chapter COVISE CE

 

With the options provided by CSCW you

  • include a remote host (AddHost) for distributed working
  • include a remote partner (AddPartner) for collaborative working
  • delete a remote host or partner (DelHost/Partner)

    • (for distributed and collaborative working see Collaborative Working)
  • mirror a whole pipeline (or part of it) to another host

    • (Set Mirrors/Mirror Nodes/Remove Mirrored Nodes)
     
  • open NS³ (Conference Room Interface) (optional)
By AddHost ...

you can introduce additional hosts at nearly every moment of a COVISE session. The purpose is to make use of specific resources such as supercomputers or fileservers. Even if the added host is another workstation, it is not added to a cooperative working session (distributed, not collaborative working).

By AddPartner...

you invite a partner to participate in a current COVISE session (collaborative working). As initiating partner you play the Master role.
 
Note: 
You can use shortcuts from the ToolBar:
CSCW > Addhost:
CSCW > AddPartner:

To add a host/partner go thru the following steps:

  • Select Addhost or AddPartner to get the window Add host/partner (see below on the left side)
  • Select a hostname from the list (taken from the HostConfig scope of the COVISE configuration file), press 'Enter'

    •  
  • If COVISE has information about  the selected host you get this info in the window Set connection parameters (see above on the right side):

    •       Hostname and Userid
          the default Execution Mode to add the host
          the default Time-out (for TCP-connections)
  • Increase timeout if your network is slow

    •  
  • Select execution mode

    •    rexec: hostname, userid, and password on remote machine required
       rsh or ssh: can be used if no password required
       nqs: not recommended, uses batch queue
       manual: if required by security reasons
       (session across firewall or no account on remote machine,
        for details see Tutorial, Advanced Topics )
When the remote computer is added successfully, an entry for this host (selection tree) will be added to the ModuleBrowser (see left picture for an example) .

In case of AddPartner a MapEditor will pop up on the remote workstation.

To choose a module from an added host, select first the corresponding hostname in the module browser and proceed then with module selection as usual (using the relevant branch of the ModuleBrowser). If specified in the COVISE configuration file, new host and modules from the host get the same color (different from your host), otherwise you get qualified module names (hostname::module name).
(colored module icons are default; if you like qualified names, please specify in the covise.config file)

DelHost/Partner is  just the inverse operation.
 

For functions available as Slave see option MasterControl

To send infomation to partners (independent from master or slave state) use the Chat Line (and receive information in the Message Area).

For general information on distributed and collaborative working see Collaborative Working, Tutorial (Advanced Topics), and User's Guide, chapter COVISE CE.
