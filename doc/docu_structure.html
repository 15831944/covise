Documentation Structure (Overview)

To anybody who wants to generate/test (or make additions/changes to) COVISE documentation after I have left VirCinity   -

- this is the current structure of COVISE documentation (update 12/10/02):

Please do not change the structure if it is not necessary and if you are not absolutely sure that you know what you are doing!!!
(Trotzdem: Vater, vergib ihnen, denn sie wissen nicht was sie tun! Lk 23,34)

Update 12/10/02: I changed  'VR and CE User's Guide' back to 'User's Guide' because 'VR and CE' had been introduced mainly for of marketing reasons - and now I  decided not to confuse the small number of customers and employees with this complicated title any longer.

General Remarks:
 

Online Help is written mostly using the Netscape Composer and - if necessary for special purposes - changing the HTML source code.

  • If you make changes in online help, use Netscape Composer!
  • If you have made changes in other documents, please check if you have to change links in online help!
  • All images in online help are .gif files - if you replace images, check if you should replace images in Tutorial or User's Guide, too.
Other documentation is written in Latex and is converted to HTML using a LatextoHTML converter.
In case of changes
  • adapt structure of the online documention to the structure of the Latex docu if necessary; this may include new chapters, new start-files, and new tables of contents or changes in existing tocs.
  • Since the converter has some restrictions/deficiencies please be careful in using other Latex features than used before, and/or use "latexonly" and "htmlonly" constructs as used especially for tables in the programming guide. One of the major restrictions of the converter is the fact that it does not preserve captions. If you want to transfer a caption to the online docu, please use a "htmlonly"-construct.
  • If you make changes in any document, please look if you have to change links in online help!
  • Images are .gif and .png for HTML and .eps for Latex; use "xv" (and change manually ".ps" to ".eps") to convert from .gif  or .png to .eps.
